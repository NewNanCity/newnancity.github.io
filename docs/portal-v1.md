# 玩家门户 V1

> 首页预算已由 `docs/portal-v2-aggregation.md` 迭代；本文件保留 V1 的路由和页面分层决策。

## Outcome

牛腩小镇主站从长瀑布展示页收敛为玩家门户：首页在固定内容预算内聚合入口与更新，完整内容进入一级、二级和三级页面。现有城镇独立站继续作为三级页面，不被统一模板取代。

## Context

- React 主站目前只有首页与 `#map` 两种状态，内容事实源是 `public/site-data.json`。
- 历史、图库、加入流程已有可复用组件；城镇独立站发布到 `/towns/**`。
- GitHub Pages 不提供应用路由回退，因此门户采用兼容直达的哈希路由。
- 服务器状态和指标 API 尚未提供，本版只保留未来数据接入位置。

## Information Architecture

```text
首页聚合
├─ 世界
│  ├─ 城镇目录
│  │  └─ 标准城镇档案
│  │     └─ 城镇独立主页
│  ├─ 实时地图
│  └─ 城镇评级
├─ 社区
│  ├─ 官方动态
│  └─ 玩家页面入口
├─ 档案
│  ├─ 六年历程
│  ├─ 小镇风光
│  └─ 牛腩精神
└─ 加入
   └─ 入服流程与外部服务
```

一级入口固定为 `世界`、`社区`、`档案`、`加入`。低频外部链接放入辅助入口，不与一级入口争夺视觉层级。

## Homepage Budget

首页只允许以下内容单元：

1. 紧凑 Hero：品牌、稳定介绍与加入入口。
2. 四个一级入口。
3. 一个世界焦点和最多三条最近更新。
4. 一条准入提示。

新内容必须进入现有插槽或一级页面，不得通过新增首页 section 扩张页面长度。动态表示数据可替换和可聚合，不要求自动轮播或实时 API。

## Routing

- `#`、`#/`：首页。
- `#/world`：世界目录。
- `#/world/towns/:slug`：标准城镇档案。
- `#/community`：社区入口。
- `#/archive`：档案页。
- `#/join`：加入页。
- `#map` 与 `#/map`：实时地图，保留旧地址兼容。
- 未识别路由：显示明确的门户内 404，不静默回退首页。

## Content Contract

`site-data.json` 新增 `portal`：

- `gateways`：四个一级入口及视觉摘要。
- `spotlight`：首页世界焦点。
- `updates`：最多三条首页更新，包含明确日期。
- `towns`：标准城镇档案数据与三级页面地址。
- `community`：已审核的社区入口；玩家自建脚本不在主站同源执行。

本版数据仍由静态 JSON 提供，并通过同一运行时/构建期校验器验证。未来 API 应在数据提供层转换为同一视图契约，页面组件不直接猜测缺失字段。

## Constraints

- 正式域名保持 `newnan.city`，现有 `/towns/**` 和 `#map` 地址向后兼容。
- 不新增路由或图标依赖；沿用现有 React、TypeScript、CSS 和哈希状态模式。
- 不公开服务器连接地址，不实现服务器状态或指标 API。
- Minecraft 场景图保留 `image-rendering: pixelated`。
- 关键内容默认可见，动效只能增强，必须遵守 `prefers-reduced-motion`。
- 玩家自建页面只通过审核后的元数据引用；任意脚本托管和同源隔离不在本版实现。

## Non-goals

- 不重写现有城镇独立站。
- 不实现玩家投稿、认证、托管或审核后台。
- 不实现全文搜索、实时通知、服务器状态和在线人数接口。
- 不提交、推送或部署生产环境。

## Acceptance Criteria

- 首页不再平铺历史、图库、精神和完整加入流程；四个一级入口首屏后即可到达。
- 四个一级入口都有可用页面，不出现纯占位路由。
- 世界页可进入至少三个标准城镇档案，再进入对应独立主页。
- `#map` 旧链接仍可用；非法哈希显示门户 404。
- 桌面与 390px 移动端无横向溢出、文字裁切、不可达交互或遮挡。
- 键盘导航、焦点样式、减少动态效果、console 和网络请求通过真实浏览器验收。
- `pnpm test`、`pnpm exec tsc -b`、`pnpm run build`、`git diff --check` 通过。

## Integration Notes

- `public/site-data.json` 是内容事实源，组件不复制城镇文案或精选顺序。
- 路由解析保持为无副作用的纯函数，并单独测试旧链接与非法输入。
- 图片继续经过现有主站/城镇发布管线；首页不得热链第三方封面。
- 城镇独立站保持视觉自治，只补充必要的返回门户路径时才修改。

## Stop Rules

- 需要改变现有城镇 URL、公开服务器地址、引入玩家脚本托管或修改部署方式时停止并确认。
- 路由方案需要服务器回退、认证或新依赖时停止并确认。
- 同一浏览器问题连续三次无法稳定复现或修复时停止并报告证据。
