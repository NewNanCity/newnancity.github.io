# 首页动态聚合 V2

## Outcome

首页在不恢复长瀑布的前提下，直接展示来自新闻、城镇、活动、历史和风景的内容碎片。浏览者无需先理解网站结构，也能看见牛腩正在发生什么、曾经发生过什么，并从任意碎片继续进入对应页面。

## Product Assumptions

- “动态”包含两个不同层面：页面内轮换展示，以及内容源随时间更新。本版实现前者，并为后者定义边界；不把静态 JSON 伪装成实时 API。
- 首页仍有明确高度预算。新增内容必须进入聚合面板、快捷入口或社区口号三个固定位置，不再增加独立长 section。
- 聚合条目必须来自已有新闻、城镇、活动、历史或图库事实；没有来源的玩家活动不得编造。
- 四个一级入口继续承担完整内容导航，聚合面板负责让访客在首页先看到有吸引力的碎片。

## Homepage Budget

```text
紧凑 Hero
└─ 温暖的入服提醒
世界播报：固定高度，1 个当前内容 + 其余条目标题
四个一级入口：世界 / 社区 / 档案 / 加入
常用入口：小镇名册 / 百科 / 皮肤站 / NewNan ID
社区口号：有你，才叫牛腩
Footer
```

“世界播报”允许 5 至 8 条内容，自动轮换但始终显示全部标题。鼠标悬停、键盘聚焦、页面不可见、用户手动暂停或 `prefers-reduced-motion` 生效时停止轮换。当前条目提供图片、摘要和明确去向，其余条目仍可直接选择，不使用无限滚动。

## Content Contract

`site-data.json` 的 `portal` 增加：

- `feed`：首页聚合条目，类型为新闻、城镇、活动、回忆或风景。
- `ecosystem`：常用生态入口，至少包含小镇名册、百科、皮肤站和 NewNan ID。

旧 `spotlight` 与 `updates` 字段暂时保留，避免静默破坏社区页和既有内容契约。所有首页图片继续进入现有分层图片管线。

## Tone Guide

语气应像熟悉服务器的老朋友在带新人逛小镇：陈述准确，但不使用产品规划、数据平台或安全系统的内部措辞。

- 不写“首页只负责指路”，改为“城镇在生长，故事也在继续”。
- 不写“安全准入”，改为“开门之前，先认识一下彼此”。
- 不写“玩家页面计划、审核与隔离方案”，改为“玩家作品正在集合”。
- 不写“标准档案”，面向玩家使用“城镇名片”“认识这座城”。
- 技术边界留在文档与代码，不作为首页功能说明展示。

## Wolai Town Source

2026-07-17 浏览器观测到 Wolai 城镇列表共有 30 条公开记录，包含名称、星级、成立时间、负责人、官网、群号、类型和备注。页面通过 Wolai 未文档化接口读取数据；该接口允许服务端请求，但浏览器 CORS 仅允许 `www.wolai.com`，因此主站不能直接在运行时安全同步。

当前不将该内部接口接入生产构建，原因如下：

- 表字段使用不透明 ID，Wolai 调整列或接口即可导致静默错配。
- 表内缺少稳定 `slug`、首页摘要、封面、是否推荐等展示字段。
- 部分官网仍为旧 `www.newnan.city` 地址，需要先在事实源修正。
- 将第三方页面可用性变成每次官网构建的硬依赖，会降低发布可靠性。

推荐的正式同步契约：

1. Wolai 增加稳定 `slug`、`summary`、`cover_url`、`featured` 和规范 `site_url` 字段。
2. 由受控脚本生成带 schema 版本的 `towns-snapshot.json`，仅在完整校验后替换上一份快照。
3. 同步失败继续使用最后一份有效快照，并明确记录失败，不让官网构建猜测字段。
4. 定时 GitHub Action 或自有 JSON API 需要单独确认部署与运维责任后再接入。

本版首页和世界页直接提供 Wolai 完整名册入口，主站继续维护少量有图片和故事的精选城镇名片。

## Constraints

- 保持 `newnan.city`、现有哈希路由、`/towns/**` 和旧 `#map` 地址兼容。
- 不新增运行时依赖，不热链第三方内容图片。
- 保留 Minecraft 场景图的 `image-rendering: pixelated`。
- 自动轮换必须可暂停，关键内容不得依赖动画后才可见。
- 不修改 Wolai、Wiki、皮肤站、NewNan ID 或 GitHub Actions 的公共契约。

## Non-goals

- 不接入服务器状态、在线人数或注册人数 API。
- 不实现玩家投稿、页面托管、认证或审核后台。
- 不把 Wolai 内部接口加入生产运行时或默认构建。
- 不提交、推送或部署生产环境。

## Acceptance Criteria

- 首页首个内容区可轮换至少五种内容碎片，其他标题始终可见且可选择。
- 自动轮换、手动选择、暂停、悬停/聚焦暂停和 reduced motion 行为可验证。
- 小镇名册、百科、皮肤站和 NewNan ID 在首页作为可见快捷入口出现。
- “有你，才叫牛腩”与社区特色回到首页下方，但页面不恢复为无上限瀑布流。
- 首页及一级页面不再展示产品规划、隔离方案、标准档案或安全准入等内部措辞。
- 桌面与 390px 移动端无横向溢出、遮挡、裁切、损坏图片、console error 或失败的主站请求。
- `pnpm test`、`pnpm exec tsc -b`、`pnpm run build`、`git diff --check` 通过。

## Stop Rules

- 需要将 Wolai 内部接口加入生产构建、修改 GitHub Actions 定时发布或引入凭证时停止确认。
- 需要更改现有路由、认证边界或第三方站点内容时停止确认。
- 聚合面板在移动端无法维持有限高度，或自动轮换无法满足可访问性要求时停止并重新评估交互形式。
